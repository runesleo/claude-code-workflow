## 📋 工作流程

### 每日站会（虚拟）
**时间**: 每工作日开始  
**参与者**: 全体团队  
**内容**:
1. 昨天完成了什么
2. 今天计划做什么
3. 遇到什么障碍

---

### 代码审查流程
```
开发提交 → 技术经理审查 → QA验证 → 总架构师批准 → 合并
```

**审查标准**:
- ✅ 代码质量
- ✅ 测试覆盖
- ✅ 文档完整
- ✅ 性能影响
- ✅ 向后兼容

---

### 决策流程
```
问题提出 → 技术经理分析 → 团队讨论 → 总架构师决策 → 执行
```

**决策类型**:
- **P0**: 立即决策（架构方向）
- **P1**: 1天内决策（技术方案）
- **P2**: 3天内决策（优化改进）

## 📝 团队协议

### 代码提交协议
```
<type>(<scope>): <subject>

<body>

<footer>
```

**类型**:
- `feat`: 新功能
- `refactor`: 重构
- `test`: 测试
- `docs`: 文档
- `chore`: 构建/工具

### 审查协议
- 24小时内完成审查
- 建设性反馈
- 关注代码质量和测试

### 文档协议
- 代码变更必须更新文档
- 文档先行（重要功能）
- 示例代码必须可运行
